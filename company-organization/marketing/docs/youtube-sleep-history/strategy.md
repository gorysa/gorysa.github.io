# 睡眠導入向けニッチ世界史チャンネル 戦略設計

## 目的
YouTube広告収入で月10万円を目指しつつ、AIを使って運営の自動化率を高める。ただし、完全放置ではなく、収益性・継続性・安全性を両立できる「半自動オペレーション」を構築する。

## 前提
- 想定ジャンルは「睡眠導入としてのニッチな世界史ナレーション」。
- 収益源の第一優先は YouTube パートナープログラムの広告収益。
- 運営方針は「低品質量産」ではなく、一次・二次資料を踏まえた独自構成、落ち着いた語り口、長時間視聴されやすい編集を重視する。
- AIは調査補助、企画補助、台本下書き、品質チェック、運用ログ整理に使う。
- 最終公開物には人の編集判断を残し、誤情報、著作権侵害、再利用コンテンツ判定のリスクを下げる。
- 不明点については以下を仮定する。
  - 1本あたりの動画尺は 25〜60 分。
  - 投稿本数は初期 週2本、安定後 週3本を上限目安とする。
  - 音声は自社制作ナレーションまたは商用利用条件を満たす音声生成を前提にする。
  - 映像素材は自作の図版・地図・字幕・簡易アニメーションを中心にし、第三者素材依存を避ける。

## 戦略

### 1. チャンネル戦略の基本方針
- 睡眠導入用途なので、クリックだけでなく「低刺激でも最後まで流しやすい」体験を作る。
- 世界史の中でも競合が比較的薄いテーマに絞る。例: 辺境交易、消えた王朝の行政制度、珍しい宗教儀礼、古代港湾都市、失われた少数国家など。
- 1本ごとに「知らない話だが安心して聞ける」構成にするため、刺激の強い演出より、静かな構成・一定テンポ・穏やかなサムネイルを優先する。
- 量産よりシリーズ化を重視する。シリーズ化により調査資産、用語集、地図テンプレート、タイトル型を再利用できる。

### 2. 収益達成の現実的な考え方
- 月10万円達成は、単発のバズ依存より「ライブラリ型」で狙う。
- 検索・関連動画・睡眠/BGM需要からの長期流入を蓄積する。
- 1本の成功でなく、一定水準以上の動画を継続投入し、既存動画群が毎月視聴時間を積み上げる構造を作る。
- そのために、MVPでは「企画の当たり外れ」より「安全に継続できる制作システム」の確立を優先する。

### 3. 差別化ポイント
- 一般教養系ではなく「睡眠導入に適した情報設計」に特化する。
- 台本は、煽り・断定・テンション高めを避け、背景説明→制度や生活描写→静かなまとめ、の流れを標準化する。
- BGM を主役にせず、声と間、語彙のやわらかさ、音量設計で睡眠導入に最適化する。
- 独自図版や地図テンプレートを内製し、再利用コンテンツ判定を避ける。

### 4. 運用モデル
- 企画、リサーチ、初稿生成、メタデータ案、ログ集計はAI中心。
- ファクトチェック、言い回し調整、公開判断は人間中心。
- 編集はテンプレート化して半自動にし、完全自動化しない。
- 毎週、制作本数よりも CTR、平均視聴時間、睡眠用途コメント、視聴者維持率を振り返る。

## KPI仮説

### 収益化前の必達KPI
- YPP参加条件の達成。
  - 仮説1: 長尺動画中心のため、公開 90 日時点で総再生時間 4,000 時間を主ルートとして狙う。
  - 仮説2: 登録者 1,000 人は、動画末尾と概要欄で「寝落ち用に保存」「次回テーマ通知のため登録」を自然に促し達成する。

### 月10万円のためのKPI仮説
以下は概算仮説であり、実測データで更新する。

| 指標 | 仮説 |
|---|---:|
| 月間再生回数 | 80,000〜180,000 回 |
| 平均動画尺 | 35〜50 分 |
| 平均視聴時間 | 12〜20 分 |
| 月間総視聴時間 | 16,000〜60,000 時間 |
| 想定RPM | 200〜800 円 |
| 想定広告収益 | 約1.6万〜14.4万円 |

### KPIの読み方
- このジャンルは視聴時間が伸びやすい一方、睡眠用途で広告単価が大きく跳ねる保証はない。そのため RPM は広めに見積もる。
- 月10万円達成の中心シナリオは「月間 120,000 回再生 × RPM 500〜900 円」ではなく、より保守的に「中尺〜長尺動画の蓄積で、月間 100,000 回前後でも届く可能性がある設計」を狙う。
- 重要なのは再生数だけでなく、視聴時間の安定蓄積と動画資産数である。

### 初期運営の先行KPI
- 1か月目
  - 投稿 8 本
  - CTR 3.0〜5.0%
  - 平均視聴時間 8 分以上
  - 30秒到達率 70%以上
- 2か月目
  - 投稿累計 16 本
  - 1本あたり再生数中央値 500〜1,500
  - 平均視聴時間 10 分以上
  - 登録転換率 0.8〜1.5%
- 3か月目
  - 投稿累計 24 本前後
  - 月間視聴時間 1,500〜4,000 時間
  - 再生上位 20% 動画の共通特徴を抽出済み

## 工程分解
1. 市場理解
   - 競合観察
   - 視聴者ニーズ仮説整理
   - サブジャンル選定
2. コンテンツ企画
   - テーマ候補収集
   - タイトル仮説作成
   - シリーズ設計
3. リサーチ
   - 信頼できる参考資料収集
   - 出典メモ作成
   - 禁止表現・危険論点確認
4. 台本制作
   - 構成案作成
   - 台本初稿生成
   - 眠りやすさ調整
   - ファクトチェック
5. 音声制作
   - ナレーション生成または収録
   - 音量/間/抑揚調整
6. ビジュアル制作
   - サムネイル
   - 地図、年表、簡易図版
   - 動画テンプレート差し込み
7. 動画編集
   - 音声と図版の同期
   - テロップ最小化
   - 書き出し
8. 公開準備
   - タイトル/説明文/タグ/チャプター
   - 公開前チェックリスト
   - 投稿予約
9. 効果測定
   - YouTube Studio 指標取得
   - 動画別レビュー
   - 仮説更新
10. ナレッジ蓄積
   - 用語集更新
   - 再利用テンプレート更新
   - 失敗パターン記録

## 自動化分類

| 工程 | 完全自動化 | 半自動化 | 人の確認必要 | 理由 |
|---|---|---|---|---|
| 競合候補の収集 | ○ |  |  | APIや手動入力データから一覧化しやすい |
| テーマ候補の発散 |  | ○ |  | AI案出しは得意だが、ブランド適合の判断が必要 |
| 出典候補の収集 |  | ○ |  | AIで候補列挙、採用は人が判断 |
| 台本構成の下書き |  | ○ |  | テンプレ化できるが独自性確認が必要 |
| 事実確認 |  |  | ○ | 誤情報リスクが高い |
| 眠りやすい語調への整形 |  | ○ |  | ルール化しやすいが最終読感は人が見る |
| ナレーション生成 | ○ |  |  | 条件が定まれば自動実行可能 |
| 音声品質判定 |  | ○ |  | ノイズ・不自然箇所の自動検出＋人確認 |
| 地図/図版のテンプレ生成 |  | ○ |  | テンプレ自動化は可能、史実整合は人確認 |
| 動画編集の基本流し込み |  | ○ |  | テンプレで省力化可能 |
| サムネイル案生成 |  | ○ |  | 量産可能だが選定は人が行う |
| タイトル/説明文案生成 |  | ○ |  | SEO と誇張回避のバランス確認が必要 |
| 公開前品質ゲート |  |  | ○ | 収益・安全性に直結 |
| KPI集計 | ○ |  |  | API/CSV連携で自動化しやすい |
| 振り返りレポート生成 |  | ○ |  | AI要約は可能、意思決定は人が行う |

### 結論
- 完全自動化してよいのは「集計」「候補生成の一部」「定型変換」に限る。
- 再利用コンテンツ判定、著作権、誤情報、ブランド毀損につながる工程は人の確認を必須にする。
- 目指すべきは「60〜75%の半自動化」であり、「100%自動運営」ではない。

## MVP案

### 最初に作るべき仕組み
1. **企画バックログ生成器**
   - 入力: サブジャンル、対象時代、避けたいテーマ、既出テーマ。
   - 出力: 動画案、仮タイトル、想定視聴者、競合差別化メモ。
2. **リサーチ・出典管理シート**
   - 各動画ごとに、主張、根拠資料、未確認事項、使用可否を記録する。
3. **睡眠導入向け台本テンプレート**
   - 導入、背景、具体描写、静かな締め、の定型構造を持つ。
4. **公開前品質ゲート**
   - 著作権、再利用、誤情報、刺激表現、音量、サムネ誇張をチェックする。
5. **KPIログ基盤**
   - 動画ごとの CTR、平均視聴時間、公開後 48 時間再生数、7日後維持率、コメント傾向を蓄積する。

### MVPでやらないこと
- フル自動動画生成パイプライン。
- ニュース系・速報系への拡張。
- 複数チャンネル同時運用。
- 外部素材を大量収集して自動編集する仕組み。

## 90日ロードマップ

### 0〜30日: 土台構築フェーズ
- チャンネルのブランド定義、禁止事項、トーン定義を決める。
- サブジャンルを 3 系統に絞る。
- 台本テンプレート、出典管理フォーマット、品質チェックリストを作る。
- まず 8 本分の企画候補を作り、上位 4 本を制作する。
- ログ基盤を先に整え、動画公開と同時に結果を記録できるようにする。

### 31〜60日: 検証フェーズ
- 週2本運用を安定化する。
- タイトル型、サムネイル型、尺の違いをA/Bではなく「週次単位の比較実験」として回す。
- 高維持率テーマの共通点を抽出し、シリーズ化候補を決める。
- 低調動画の原因を、テーマ、導入30秒、音声、サムネイルに分解して記録する。

### 61〜90日: 最適化フェーズ
- 上位テーマ群へ集中し、シリーズを 2 本柱化する。
- 台本生成とメタデータ生成のルールを固定化し、制作時間を削減する。
- 投稿後 7 日・28 日レビューを標準化する。
- 90 日終了時に、継続投資すべきフォーマットと捨てるフォーマットを明確にする。

### ログと検証方法
- 各動画に一意の `video_id` を付与する。
- 企画時点から公開後レビューまで、同じ `video_id` でログをつなぐ。
- 最低限残すログ:
  - 企画採用理由
  - 出典URL/書籍情報
  - 台本生成プロンプト版数
  - 人による修正箇所
  - 公開設定
  - 48時間/7日/28日指標
- 検証単位は「動画単体」と「シリーズ単位」の両方で見る。
- 成功判定は単純な再生数ではなく、CTR、平均視聴時間、維持率、コメントの質、制作時間、修正回数で行う。

## リスク一覧

| リスク | 内容 | 回避策 |
|---|---|---|
| 著作権侵害 | 画像・音声・BGM・地図素材の権利問題 | 自作または商用利用条件を確認した素材のみ使用 |
| 再利用コンテンツ判定 | AI量産感、他動画との差別化不足 | 独自台本、独自構成、自作図版、手動編集を残す |
| 誤情報 | 世界史は解釈差や史料差がある | 出典管理、断定表現の抑制、公開前レビュー |
| 低品質量産 | 自動化を優先しすぎて視聴体験が劣化 | 品質ゲートを通らない動画は公開しない |
| 音声の不自然さ | TTSが単調/不自然で離脱を招く | サンプル比較、句読点調整、人の試聴確認 |
| クリックは取れるが睡眠用途に不向き | タイトルや導入が刺激的すぎる | 睡眠導入ブランドガイドを設定 |
| データ不足で誤学習 | 初期に少数動画だけで結論を出す | 8〜12本以上で傾向判断 |
| 制作負荷の偏り | 調査だけ人力負担が大きい | サブジャンル固定、用語集蓄積、出典テンプレ利用 |
| プラットフォーム依存 | YouTubeアルゴリズム変動 | メール/ブログ/再生リスト導線の整備を検討 |
